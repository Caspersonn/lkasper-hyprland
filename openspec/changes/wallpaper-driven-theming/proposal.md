## Why

Every UI colour in the setup is static and duplicated: the AGS shell hardcodes a gruvbox palette (plus the `$w-*` "Wood Dark" tokens from the recent bar work), while the terminal, btop, hyprlock and Hyprland borders derive from a single `nix-colors` base16 `colorScheme` — and the AGS shell is the one component *not* wired into it. There is no way to make the desktop follow the wallpaper. We want the wallpaper to drive every colour (bar, overlays, popups, and the rest of the desktop), selected through a picker, with no hardcoded colours anywhere.

## What Changes

- Derive a **base16 palette per wallpaper** with `wallust`, generated by an explicit `regenerate-palettes` command and **committed** to the repo (reproducible, no Import-From-Derivation).
- **Move the static wallpapers** (`.png`/`.jpg`) into this repo under `wallpapers/`, colocated with the theming; `hyprpaper` points at the in-repo copy. Video wallpapers (`.mp4`/`.gif`) stay in `lkasper-flake` and are out of scope (a later change, likely mpvpaper + frame-extract).
- Route the **whole desktop off the active wallpaper's palette** — the existing `nix-colors` `colorScheme` becomes wallpaper-derived rather than a hardcoded `"gruvbox"` pick, so terminal / btop / hyprlock / Hyprland borders follow automatically.
- Wire the **AGS shell into the palette** and have it consume colours at **runtime** so it recolours live on a wallpaper switch. **BREAKING**: retires the hardcoded gruvbox `$…` vars and the static `$w-*` tokens in `ags/style.scss`; AGS's colour layer moves off Sass variables (the `de2q` "colouring standard" cleanup).
- Add a **wallpaper picker** — an AGS overlay showing thumbnails of `wallpapers/`, opened by a keybind (and/or bar button); selecting one runs the switch.
- Add a **`theme-switch`** action: set the wallpaper via hyprpaper IPC, repoint `~/.config/lkasper-hyprland/current/theme.name`, and reload each app (Hyprland borders via `hyprctl`, AGS via `App.apply_css`, terminals/btop via their reload path).
- **Semantic colours (ok / warn / crit) become wallpaper-derived** too (base16 `base0B` / `base0A` / `base08`) — no static colours remain.

## Capabilities

### New Capabilities
- `wallpaper-palette-generation`: derive and commit a base16 palette for each wallpaper in `wallpapers/`, via a repeatable generator command; Nix consumes the committed palettes.
- `wallpaper-theme-switching`: at runtime, selecting a wallpaper sets it and recolours the whole desktop from that wallpaper's palette — including the AGS shell consuming the palette live — with a defined per-app reload behaviour and a fallback when generation/data is missing.
- `wallpaper-picker`: an AGS overlay that lists the available wallpapers as thumbnails and lets the user pick one (keyboard + click), triggering the switch.

## Impact

- **New dependency**: `wallust` (base16-native palette generator, in nixpkgs).
- **Nix modules**: `modules/_themes.nix` (registry becomes wallpaper-derived), `modules/home-manager/themes.nix` (palette source + runtime files + reload wiring), `modules/home-manager/hyprpaper.nix` (runtime wallpaper switching via IPC), `modules/home-manager/_hyprland/bindings.nix` (picker keybind).
- **AGS shell** (`ags/`): new `windows/wallpaper-picker/` overlay; `style.scss` colour layer refactored to a runtime-injected palette; `app.ts` wiring for live recolor; retires `$w-*` and gruvbox vars.
- **Generated/committed artifacts**: one base16 palette file per wallpaper under the repo; `regenerate-palettes` command.
- **Related work**: implements bean `lkasper-hyprland-9ura`; subsumes bean `de2q` (default colouring standard) for AGS; supersedes the static palette introduced during the bar redesign.
- **Wallpapers**: the static set moves from `~/lkasper-flake/wallpapers/` into this repo's `wallpapers/`; `lkasper-flake`'s hyprpaper path is updated accordingly.
- **Non-goals**: video wallpapers (`.mp4`/`.gif`); arbitrary runtime images without a repo palette (still require running `regenerate-palettes` + commit); animated/transition recolouring.
